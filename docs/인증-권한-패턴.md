# 인증·권한 패턴 가이드

## 1. @IfLogin (선택적 인증)

비로그인 사용자도 접근 가능하되, 로그인 시 추가 정보를 활용할 때 사용합니다.

```java
@GetMapping("/matches")
public ApiResponse<MatchListResponse> getMatches(
        @IfLogin UserPrincipal principal,
        @RequestParam(required = false) String regionCode) {
    // 비로그인: principal == null, regionCode 쿼리 파라미터 사용
    // 로그인: principal != null, interestLoc1/2를 기본 regionCode로 적용 가능
    String region = principal != null && principal.getInterestLoc1() != null
            ? principal.getInterestLoc1()
            : regionCode;
    // ...
}
```

- **SecurityConfig**: 해당 경로는 `permitAll()` (예: `/api/matches` → `/api/**`)
- **Resolver**: 로그인 시 `UserPrincipal`, 비로그인 시 `null` 주입

---

## 2. @AuthenticationPrincipal (인증 필수)

로그인이 반드시 필요한 API입니다.

```java
@GetMapping("/me")
public ApiResponse<ProfileResponse> getMyProfile(
        @AuthenticationPrincipal UserPrincipal principal) {
    Long userId = AuthUtils.getUserIdOrThrow(principal);
    // ...
}
```

- **SecurityConfig**: 해당 경로는 `authenticated()`
- **AuthUtils.getUserIdOrThrow**: principal이 null이면 UNAUTHORIZED 예외

---

## 3. @PreAuthorize (Role 기반 접근 제어)

사용자 Role에 따라 메서드 실행 여부를 결정합니다.

```java
// USER 역할만 접근 가능
@PreAuthorize("hasRole('USER')")
@PostMapping("/matches")
public ApiResponse<MatchResponse> createMatch(...) { }

// ADMIN 역할만 접근 가능
@PreAuthorize("hasRole('ADMIN')")
@DeleteMapping("/admin/users/{userId}")
public ApiResponse<Void> deleteUser(...) { }

// USER 또는 ADMIN 모두 접근 가능
@PreAuthorize("hasAnyRole('USER', 'ADMIN')")
@GetMapping("/reports")
public ApiResponse<ReportResponse> getReports(...) { }
```

- **@EnableMethodSecurity**: SecurityConfig에 적용됨
- **UserPrincipal**: `Role.USER` 기본값, JwtAuthenticationFilter에서 Role 전달 가능
- **Role 확장**: User 엔티티에 role 컬럼 추가 후, JwtAuthenticationFilter에서 `new UserPrincipal(userId, user.getRole())` 로 주입
