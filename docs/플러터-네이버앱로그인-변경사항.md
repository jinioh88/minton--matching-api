# Flutter 네이버/구글 앱 로그인 변경사항 정리

## 목적

Flutter에서 `flutter_web_auth_2` 기반 네이버/구글 Authorization Code 로그인 연동을 위해, 백엔드에 필요한 브리지 콜백/보안(state)/설정 검증 로직을 추가했다.

---

## 변경 요약

- 네이버 로그인은 기존과 동일하게 Authorization Code 기반 유지
- 구글 로그인도 동일한 모바일 브리지 구조로 확장
- 커스텀 스킴 직접 콜백 대신, 브리지 콜백 URL(`https/http`)을 통해 앱 스킴으로 302 리다이렉트
- OAuth `state`를 서버에서 발급·검증하도록 추가(네이버/구글 공통, 서명 + 만료시간)
- `/api/auth/oauth/login` 실패 시 provider/원인 구분이 가능한 메시지 강화

---

## 신규/변경 API

### 1) 네이버 state 발급 API (신규)

- **GET** `/api/auth/oauth/naver/state?provider=NAVER`
- 응답 예시

```json
{
  "success": true,
  "data": {
    "state": "base64url.payload.signature",
    "expiresInSeconds": 300
  }
}
```

- 용도: Flutter가 네이버 인가 요청 전에 CSRF 방지용 state 확보

### 2) 네이버 브리지 콜백 API (신규)

- **GET** `/oauth/naver/callback?code=...&state=...`
- 동작:
  1. `code`/`state` 필수값 검증
  2. `state` 서명/만료/provider 검증
  3. 성공 시 `com.mintonmatch.app://oauth/callback?code=...&state=...` 로 `302`

### 2-1) 구글 브리지 콜백 API (신규)

- **GET** `/oauth/google/callback?code=...&state=...`
- 동작:
  1. `code`/`state` 필수값 검증
  2. `state` 서명/만료/provider 검증(`Provider.GOOGLE`)
  3. 성공 시 `com.mintonmatch.app://oauth/callback?code=...&state=...` 로 `302`

### 3) OAuth 로그인 API (기존 유지, 동작 보강)

- **POST** `/api/auth/oauth/login`
- Request body
  - `provider`: `NAVER`
  - `authorizationCode`: 앱이 최종 수신한 code
  - `redirectUri`: 인가 요청에 사용했던 redirect URI와 **동일 값**
- 주의: 서버는 `redirectUri`를 임의 치환하지 않고 요청값을 그대로 토큰 교환에 사용

### 4) 구글 state 발급 API (신규)

- **GET** `/api/auth/oauth/google/state?provider=GOOGLE`
- 응답 예시

```json
{
  "success": true,
  "data": {
    "state": "base64url.payload.signature",
    "expiresInSeconds": 300
  }
}
```

## 설정 변경

`application.yml` / `application-prod.yml` 기준으로 아래 설정이 추가되거나 분리되었다.

- `app.oauth2.mobile-callback-uri`
  - 기본값: `com.mintonmatch.app://oauth/callback`
- `app.oauth2.state-signing-secret`
  - state 서명 시크릿(운영 필수)
- `app.oauth2.state-ttl-seconds`
  - state 만료 시간(기본 300초)
- `app.oauth2.allowed-redirect-uris`
  - 네이버 브리지 URI 포함
  - 예: `${NAVER_BRIDGE_REDIRECT_URI:https://api.example.com/oauth/naver/callback}`
  - 구글 브리지 URI 포함
  - 예: `${GOOGLE_BRIDGE_REDIRECT_URI:https://api.example.com/oauth/google/callback}`
- `spring.security.oauth2.client.registration.naver.redirect-uri`
  - 운영에서 `NAVER_OAUTH_REDIRECT_URI`로 분리 가능

## Flutter 연동 권장 순서

1. 앱에서 provider별 state API 호출 (`/api/auth/oauth/naver/state` 또는 `/api/auth/oauth/google/state`)
2. 수신한 `state`와 provider별 브리지 redirect URI로 인가 요청
3. OAuth 제공자가 브리지 콜백(`/oauth/naver/callback` 또는 `/oauth/google/callback`)으로 code/state 전달
4. 서버가 검증 후 앱 스킴으로 302 리다이렉트
5. 앱이 `POST /api/auth/oauth/login` 호출
   - `provider=NAVER` 또는 `GOOGLE`
   - `authorizationCode=<code>`
   - `redirectUri=<브리지 redirect URI>`

---

## 오류 처리/디버깅 개선

- 토큰 교환 실패 시 서버 로그에 아래를 남김
  - provider, HTTP status, redirectUri, tokenUri
  - OAuth 에러 코드(`invalid_client`, `invalid_grant` 등)
  - `error_description`, 응답 body
- 클라이언트 응답은 기존 `ErrorResponse` 스키마(`code`, `message`) 유지
- `message`에 provider별 실패 원인을 포함해 앱 디버깅 용이성 향상
