# 개발 환경 설정

## 사전 요구사항

- JDK 21
- Gradle 9.x

## 환경 구분

| 프로파일 | DB | 용도 |
|----------|-----|------|
| **local** (기본) | H2 (파일 기반) | 로컬 개발 |
| **prod** | MySQL (AWS RDS) | 운영 |

## 로컬 환경 (local)

- **DB**: H2 파일 기반 (`~/minton_match`에 저장, 서버 재시작 시 유지)
- **실행**: 프로파일 미지정 시 기본 적용

```bash
./gradlew bootRun
```

또는 명시적으로:

```bash
./gradlew bootRun --args='--spring.profiles.active=local'
```

### OAuth2 (카카오/네이버/구글 로그인 테스트)

로컬에서 OAuth 로그인을 테스트하려면 **환경변수**를 설정해야 합니다. (기본값 `local-placeholder`는 카카오 API에서 거부됨)

```bash
# 터미널에서 실행 시
export KAKAO_CLIENT_ID="카카오_개발자_콘솔에서_발급한_REST_API_키"
export KAKAO_CLIENT_SECRET="카카오_개발자_콘솔에서_발급한_Client_Secret"
./gradlew bootRun
```

또는 한 줄로:

```bash
KAKAO_CLIENT_ID=your_client_id KAKAO_CLIENT_SECRET=your_client_secret ./gradlew bootRun
```

**카카오 개발자 콘솔**: https://developers.kakao.com → 내 애플리케이션 → 앱 키 → REST API 키, Client Secret

### OAuth2 Redirect URI 구분 (Web / Flutter)

클라이언트별 OAuth 흐름이 다르므로 Redirect URI를 분리해서 관리해야 합니다.

- **Web(React)**
  - 로컬: `http://localhost:3000/oauth/callback`
  - 로컬 대체: `http://127.0.0.1:3000/oauth/callback`
- **Flutter(Android/iOS)**
  - 앱 딥링크: `com.mintonmatch.app://oauth/callback`

백엔드 `POST /api/auth/oauth/login` 호출 시, 클라이언트는 실제로 획득한 `authorizationCode`와 함께 본인 흐름에 맞는 `redirectUri`를 전달해야 합니다.

- Web 예시
  - `provider=KAKAO`
  - `redirectUri=http://localhost:3000/oauth/callback`
- Flutter 예시
  - `provider=KAKAO`
  - `redirectUri=com.mintonmatch.app://oauth/callback`

카카오 개발자 콘솔에도 Web/Flutter Redirect URI를 모두 등록해야 하며, 백엔드 `app.oauth2.allowed-redirect-uris` 설정값과 동일해야 합니다.

### 네이버 Flutter 브리지 콜백

네이버 콘솔은 커스텀 스킴(`com.mintonmatch.app://...`) 등록 제약이 있어 브리지 콜백을 사용합니다.

- 브리지 엔드포인트: `GET /oauth/naver/callback`
- 동작:
  - `code`, `state`를 수신
  - `com.mintonmatch.app://oauth/callback?code=...&state=...` 로 `302` 리다이렉트

설정 예시:

```bash
export NAVER_BRIDGE_REDIRECT_URI="https://{domain}/oauth/naver/callback"
export MOBILE_OAUTH_CALLBACK_URI="com.mintonmatch.app://oauth/callback"
export OAUTH_STATE_SIGNING_SECRET="충분히_긴_랜덤_시크릿"
export OAUTH_STATE_TTL_SECONDS="300"
```

앱에서 `/api/auth/oauth/login` 호출 시에는 **인가 요청에 사용한 redirectUri와 동일한 값**을 전달해야 합니다.
(예: `https://{domain}/oauth/naver/callback`)

네이버 인가 요청 전에 state를 백엔드에서 발급받아 사용하세요.

- state 발급 API: `GET /api/auth/oauth/naver/state?provider=NAVER`
- 응답: `data.state`, `data.expiresInSeconds`
- 브리지 콜백 `GET /oauth/naver/callback`에서 state 서명/만료를 검증합니다.

### H2 콘솔

- URL: http://localhost:8080/h2-console
- JDBC URL: `jdbc:h2:file:~/minton_match` (`application-local.yml`과 동일)
- User: `sa`, Password: (비워두기)

## 운영 환경 (prod)

- **DB**: AWS RDS (MySQL)
- **실행**:

```bash
./gradlew bootRun --args='--spring.profiles.active=prod'
```

### 환경변수 (운영 배포 시)

| 변수 | 설명 |
|------|------|
| DB_URL | (선택) 전체 JDBC URL. 미설정 시 기본 RDS URL 사용 |
| DB_USERNAME | DB 사용자명 |
| DB_PASSWORD | DB 비밀번호 |
