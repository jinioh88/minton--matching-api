## Phase 2: 사용자 관리 (User Management)

### Step 2.1: User 엔티티 및 기본 API
- [ ] User 엔티티 설계
  - 닉네임, 프로필 사진, 성별, 나이, 자기소개
  - 급수(A~D, 초심), 주력 라켓, 선호 플레이 스타일
  - 관심 지역(최대 2곳)
- [ ] User CRUD API (생성, 조회, 수정)
- [ ] 다른 사용자 프로필 조회 API

### Step 2.2: 인증 (로그인)
- [ ] 로그인 API (MVP: 이메일/비밀번호)
- [ ] JWT 또는 세션 기반 인증
- [ ] 소셜 로그인(카카오, 네이버, 구글, 애플) - 후순위

---

## Phase 3: 매칭 핵심 (Matching Core)

### Step 3.1: Match 엔티티 및 매칭 생성
- [ ] Match 엔티티 설계
  - 제목, 상세 설명, 장소, 일시
  - 모집 인원(최소/최대), 희망 급수
  - 코트비 분담 방식(1/N, 방장부담, 게스트부담)
  - 방장(User) 연관
- [ ] 매칭 생성 API (POST)
- [ ] 매칭 이미지 업로드 - 선택

### Step 3.2: 매칭 목록 조회
- [ ] 매칭 목록 API (GET)
  - 관심 지역 기반 필터
  - 날짜(오늘/내일), 급수, 시간대 필터
  - 페이징

### Step 3.3: 매칭 상세 조회
- [ ] 매칭 상세 API (GET)
  - 방장 프로필 요약, 후기 점수
  - 확정 인원, 대기열 현황
